How to use mill
1

Export to CSV

Pull all claims into a spreadsheet for analysis, stakeholder review, or compliance documentation.

$ mill export --format csv claims.json
2

Export to PDF

Turn your HTML brief into a polished PDF you can email or attach to a ticket.

$ mill export --format pdf output/brief.html
3

Publish static site

Generate a complete static site from your sprint output. Deploy to GitHub Pages or any host.

$ mill publish --target static output/
Real example

Exporting a brief to PDF and CSV

You have finished a wheat sprint about database migration. Now you need artifacts for the architecture review meeting.

$ mill export --format pdf output/brief.html Converting output/brief.html to PDF... Written: output/brief.pdf (142KB, 4 pages) $ mill export --format csv claims.json Extracting 23 claims... Written: claims.csv Columns: id, type, claim, evidence_tier, confidence, source, superseded_by 23 rows, ready for Excel / Google Sheets $ mill convert --from html --to markdown output/brief.html Written: output/brief.md Clean markdown with frontmatter. Paste into Notion, Confluence, or GitHub. $ mill publish --target static output/ Static site generated in dist/ 3 pages, 1 index, all assets inlined. Deploy anywhere.
Key features

Every format you need

One tool for all output formats. No config files, no build pipeline.

PDF Export

HTML and Markdown to PDF via npx md-to-pdf. Styled, paginated, ready for email or print.

CSV Export

Claims to CSV with all fields: type, evidence tier, confidence, source. Open in any spreadsheet app.

JSON-LD

Semantic web export for linked data. Machine-readable claims with full provenance.

Static Sites

Generate deployable sites from sprint output. All assets inlined, deploy to GitHub Pages or S3.
